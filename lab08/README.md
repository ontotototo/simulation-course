# Лабораторная работа 8
## Моделирование простейшего (пуассоновского) потока событий

### 1. Цель работы
Разработать программную модель простейшего пуассоновского потока заявок на сервер. Реализовать генерацию моментов наступления событий, выполнить статистическую обработку результатов (подсчет числа заявок на фиксированных интервалах). 

### 2. Математическая модель
#### 1. Описание процесса
Моделируемый процесс представляет собой последовательность событий (поступление заявок), наступающих в случайные моменты времени $t_0, t_1, t_2, \dots$. Интервалы между соседними событиями $\tau_i = t_i - t_{i-1}$ являются независимыми случайными величинами, имеющими экспоненциальное распределение с параметром $\lambda$ (интенсивность потока).

Количество событий $X(T)$, происходящих на фиксированном интервале времени длины $T$, подчиняется закону распределения Пуассона:
$$ P\{X(T) = k\} = \frac{(\lambda T)^k}{k!} e^{-\lambda T}, \quad k = 0, 1, 2, \dots $$
где математическое ожидание и дисперсия равны:
$$ E[X(T)] = D[X(T)] = \lambda T $$

#### 2. Алгоритм генерации потока
Для получения моментов событий используется метод обратного преобразования. Поскольку функция распределения экспоненциальной величины $F_\tau(x) = 1 - e^{-\lambda x}$, то обратная функция позволяет выразить интервал $\tau$ через равномерно распределенную случайную величину $\alpha \sim U(0,1)$:
$$ \tau = -\frac{\ln \alpha}{\lambda} $$
Момент наступления $i$-го события вычисляется рекуррентно:
$$ t_i = t_{i-1} + \tau_i $$

#### 3. Статистическая обработка
Для проверки соответствия потока закону Пуассона общее время моделирования $T_{sim}$ разбивается на $N$ равных непересекающихся интервалов (окон) длиной $T$:
$$ N = \lfloor \frac{T_{sim}}{T} \rfloor $$
Для каждого окна подсчитывается количество попавших в него событий $k_j$ ($j=1 \dots N$). На основе массива $\{k_j\}$ строятся:
*   Эмпирическая гистограмма частот появления $k$ заявок.
*   Выборочное среднее $\bar{k} = \frac{1}{N}\sum k_j$ и выборочная дисперсия $D_k = \frac{1}{N-1}\sum (k_j - \bar{k})^2$.

Согласно закону больших чисел, при $N \to \infty$ эмпирические характеристики должны сходиться к теоретическим значениям $\lambda T$.

### 3. Ключевые методы реализации

**Генерация потока (`run_simulation`)**
Вместо многократного перезапуска таймера, программа генерирует непрерывный поток событий на всем промежутке $[0, T_{sim}]$.
1.  Инициализируется время $t=0$.
2.  В цикле `while t < T_sim` генерируется интервал $\tau = -\ln(\alpha)/\lambda$.
3.  Время обновляется $t = t + \tau$. Если $t < T_{sim}$, момент события сохраняется в список `events`.

**Биннинг данных (Нарезка на интервалы)**
После генерации всех событий выполняется их распределение по окнам длиной $T$:
1.  Вычисляется количество окон: `num_inters = int(T_sim // T)`.
2.  Создается массив счетчиков `counts` нулевой длины `num_inters`.
3.  Для каждого момента события `ev_time` определяется индекс окна: `idx = int(ev_time // T)`.
4.  Счетчик в соответствующем окне увеличивается: `counts[idx] += 1`.
Этот подход обеспечивает эффективный сбор статистики без накладных расходов на повторную инициализацию генератора.

**Расчет статистики и визуализация**
На основе массива `counts`:
1.  С помощью `numpy.mean` и `numpy.var` вычисляются эмпирическое среднее и дисперсия.
2.  С помощью `collections.Counter` подсчитываются частоты встречаемости каждого значения $k$ (числа заявок).
3.  Частоты нормируются на общее число окон $N$ для получения эмпирических вероятностей.
4.  Метод `plot_distribution` строит гистограмму этих вероятностей.

## 5. Общий вывод по всем экспериментам:

1. Подтверждение закона Пуассона: 
- Во всех случаях, где число окон N достаточно велико (≥100), форма гистограммы соответствует ожидаемому распределению Пуассона — несимметричная при малых λT, стремящаяся к симметричной (нормальной) при больших λT.

2. Зависимость от параметров:
- Увеличение λ или T сдвигает пик распределения вправо (увеличивается среднее число заявок).
- Увеличение λT делает распределение более симметричным и «гладким».

3. Влияние объема выборки (N):
- При малом N (например, 10) график «шумный», статистика ненадёжна.
- При большом N (≥1000) график гладкий, среднее и дисперсия точно попадают в теоретические значения.
- Это иллюстрирует закон больших чисел: чем больше испытаний, тем ближе эмпирические оценки к теоретическим.

4. Равенство среднего и дисперсии:
- Во всех экспериментах эмпирическое среднее и дисперсия близки друг к другу и к значению λT.
- Например, в эксп. 5: E[k]=49.63, D[k]=51.09, λT=50 — расхождение менее 2.2%.
- Это ключевое свойство пуассоновского распределения, которое успешно воспроизведено в модели.
